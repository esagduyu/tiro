# Tiro Desktop (macOS)

The desktop app is a thin [Tauri v2](https://tauri.app) native window that manages
the PyInstaller-frozen Tiro server as a **sidecar** child process. There is no
Tauri-native UI: the window loads the normal Tiro web app over
`http://127.0.0.1:<port>`, exactly as a browser would. All app logic stays in the
Python server; the shell only owns the window, the menu, and the sidecar's lifecycle.

Two build stages:

1. `pyinstaller/` — freezes the full FastAPI server into `dist/tiro-server/` (onedir).
   This is Phase 5 / M5.0; see `pyinstaller/tiro-server.spec` and `pyinstaller/smoke.sh`.
2. `tauri/` — the native shell that bundles that frozen server as a resource and
   runs it. This is Phase 5 / M5.1 (spec D7).

> **Beta, unsigned.** 0.7.0 ships an **unsigned** local build. macOS Gatekeeper will
> refuse it on first open — right-click the app → **Open** → **Open** to run it.
> Signing + notarization is an owner-gated runbook item (spec D9), not built here.

## Toolchain prerequisites

| Tool | How | Notes |
|------|-----|-------|
| Python 3.11+ / `uv` | already required by Tiro | builds the frozen server |
| `pyinstaller` | `uv sync --group packaging` | packaging-group dep, never runtime |
| Rust (stable) + Cargo | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh -s -- -y` then `source ~/.cargo/env` | compiles the shell |
| Node + npm | Homebrew / nodejs.org | provides the Tauri CLI |
| `@tauri-apps/cli` | `cd desktop/tauri && npm install` | pinned in `package.json` |
| Xcode Command Line Tools | `xcode-select --install` | Apple linker/SDK |

Verify: `rustc --version && cargo --version && node --version && npx tauri --version`.

## Build

### 1. Freeze the server (produces `pyinstaller/dist/tiro-server/`)

```bash
# from the repo root
uv sync --group packaging
cd desktop/pyinstaller
uv run pyinstaller --noconfirm --clean tiro-server.spec
./smoke.sh           # optional but recommended: proves the binary boots + ingests offline
```

This bundle is ~830 MB (torch + ChromaDB + the seeded embedding model dominate — reported,
not fought, per spec D1). The `.spec` bundles the `all-MiniLM-L6-v2` snapshot and seeds it
into the HF cache on first launch so the very first run is offline-capable.

### 2. Build the `.app` (embeds the frozen server as a resource)

```bash
cd desktop/tauri
npm install                       # first time only
npx tauri build --bundles app,dmg # release .app + .dmg (unsigned)
# or, faster, app only:
npx tauri build --debug --bundles app
```

Output:
- `.app` → `src-tauri/target/{debug,release}/bundle/macos/Tiro.app` (~1.1 GB — it embeds
  the ~830 MB frozen server plus the WebKit shell).
- `.dmg` → `src-tauri/target/release/bundle/dmg/Tiro_0.7.0_aarch64.dmg` (when the `dmg`
  target is requested).

The app's bundle identifier is `app.tiro.desktop`. Icons are generated by
`npx tauri icon icon-master.png -o src-tauri/icons` from a 1024 px Tironian-et master
(same Pillow brand pipeline as `scripts/make_icons.py`); re-run that if the master changes.

## How it runs (spec D7)

On launch the Rust shell (`src-tauri/src/main.rs`):

1. **Config** — resolves the config path (`TIRO_CONFIG` env override, else the macOS
   platform default `~/Library/Application Support/Tiro/config.yaml`); if absent it writes
   a minimal bootstrap config (`library_path` = that dir, `host: 127.0.0.1`) at mode `0600`.
   (This is initial file *creation*; the Python side still only ever *updates* config through
   `persist_config`.)
2. **Port** — prefers the config's `port` (default 8000, keeping the Chrome extension's
   hardcoded `localhost:8000` working); if 8000 is occupied it binds `127.0.0.1:0` for a free
   ephemeral port. When the fallback engages the extension can't reach the app — a documented
   caveat; extension port configurability is future work.
3. **Sidecar** — spawns `Resources/tiro-server/tiro-server` with `TIRO_CONFIG` / `TIRO_PORT` /
   `TIRO_HOST=127.0.0.1`, in its own process group, logging to `<config dir>/sidecar.log`.
4. **Health poll** — polls `GET /healthz` for up to 60 s (warm boot ~4 s; a genuinely cold
   first boot can take ~37 s while torch/ChromaDB page in and the model cache seeds), then
   navigates the window to `http://127.0.0.1:<port>/`. A boot failure or timeout shows an
   in-window error naming the log path.
5. **Quit** — on window-close or app-quit the shell kills the sidecar **and its process
   group** (so uvicorn + the `multiprocessing` resource-tracker helper die too — no orphans).

**Menu:** About Tiro (shows the version) · Preferences… (`⌘,`, navigates the window to
`/settings`) · Quit Tiro (`⌘Q`), plus a standard Edit menu for copy/paste in the webview.

## Dev loop

`tauri dev` still needs a running server. Point the shell at the already-built frozen binary
(or any `tiro-server` executable) via `TIRO_SERVER_BIN`, and at a scratch config via
`TIRO_CONFIG`, so you never touch your real library:

```bash
cd desktop/tauri
export TIRO_SERVER_BIN="$PWD/../pyinstaller/dist/tiro-server/tiro-server"
export TIRO_CONFIG="/tmp/tiro-dev/config.yaml"   # scratch — not ~/Library/Application Support/Tiro
mkdir -p /tmp/tiro-dev
npx tauri dev
```

Rust unit tests (port pick, config bootstrap, port parsing) run locally, not in CI:

```bash
cd desktop/tauri/src-tauri && cargo test
```

## Layout

```
desktop/
  pyinstaller/           # M5.0 — freezes the server
    tiro-server.spec
    entry.py
    smoke.sh
  tauri/                 # M5.1 — the native shell
    package.json         # pins @tauri-apps/cli
    icon-master.png      # 1024px brand master for `tauri icon`
    dist-placeholder/    # loading page shown until /healthz answers
    src-tauri/
      Cargo.toml
      build.rs
      tauri.conf.json    # window + bundle + sidecar-as-resource config
      capabilities/
      icons/             # generated by `tauri icon`
      src/main.rs        # launch sequence + sidecar lifecycle
```

Build outputs (`node_modules/`, `src-tauri/target/`, `src-tauri/gen/`) are gitignored.
